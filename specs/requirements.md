# Ampersand 세션 신청 사이트 요구사항

## 목표

Ampersand 학회의 세션 신청을 온라인으로 처리한다.
회원들이 정해진 일시에 열리는 세션 목록을 보고 원하는 세션을 신청하거나 취소할 수 있으며,
모든 신청/취소 내역은 노션에 자동 기록된다.

## 사용자 역할

- 회원(일반 사용자): 세션 목록 조회, 이름과 비밀번호로 세션 신청 및 취소
- 운영진(관리자): 프로젝트 코드에서 세션 목록을 직접 수정하여 배포

## 화면 목록

### 화면 1: 메인 페이지 (세션 목록)

- 열리는 세션 전체 목록이 카드 형태로 표시된다
- 각 세션 카드에는 세션명, 발표자, 일시, 장소, 간단한 설명, 현재 신청 인원이 표시된다
- 각 세션 카드에 신청 버튼이 있다
- 신청 인원이 실시간으로 반영된다 (Supabase Realtime 활용)
- 본인이 신청한 세션은 시각적으로 구분되어 표시된다 (예: 버튼 색상 변경, "신청 완료" 표시)
- 세션 일시 기준으로 정렬하여 표시한다

### 화면 2: 세션 신청 모달

- 세션 카드의 신청 버튼 클릭 시 모달이 열린다
- 이름과 비밀번호 입력 필드가 있다
- 비밀번호는 나중에 취소 시 본인 확인용으로 사용된다
- 신청 완료 시 모달이 닫히고 신청 인원이 즉시 업데이트된다
- 동일한 이름으로 같은 세션에 중복 신청은 불가하다

### 화면 3: 세션 취소 모달

- 본인이 신청한 세션 카드의 "취소" 버튼 클릭 시 모달이 열린다
- 이름과 비밀번호를 입력해 본인 확인 후 취소한다
- 비밀번호가 일치하지 않으면 취소되지 않고 에러 메시지를 표시한다
- 취소 완료 시 신청 인원이 즉시 업데이트된다

### 화면 4: 신청자 현황 페이지 (선택)

- 세션별 신청자 명단을 확인할 수 있는 별도 페이지
- 전체 세션의 신청자 목록이 표시된다
- URL을 알면 누구나 접근 가능하다 (비밀번호 잠금 없이)

## 세션 데이터 관리

- 세션 목록은 프로젝트 내 데이터 파일(예: `src/data/sessions.ts`)에 하드코딩한다
- 세션 추가/수정/삭제는 해당 파일을 직접 편집 후 Vercel에 재배포하는 방식으로 운영한다
- 세션 데이터 항목: 세션 ID, 세션명, 발표자, 일시, 장소, 설명, 최대 수용 인원(선택)

## 노션 자동 기록

- 회원의 신청 또는 취소 발생 시 Notion API를 통해 지정된 노션 데이터베이스에 자동으로 기록한다
- 기록 항목: 이름, 세션명, 신청/취소 구분, 처리 시각
- 노션 기록 실패가 신청/취소 동작을 막지 않는다 (노션은 보조 로그 역할)

## 제약사항

- 별도 회원가입/로그인 시스템 없이 이름 + 비밀번호만으로 신청/취소한다
- 비밀번호는 평문 또는 단순 해시로 저장한다 (복잡한 인증 불필요)
- 세션 목록은 운영진이 코드에서 직접 관리하며 관리자 UI는 제공하지 않는다
- 동일 이름으로 같은 세션에 중복 신청 불가
- 신청 인원 제한이 있는 경우 초과 신청 불가

## 성공 기준

- [ ] 세션 목록 페이지가 정상적으로 표시된다
- [ ] 이름과 비밀번호 입력 후 세션 신청이 완료된다
- [ ] 신청 완료 후 신청 인원이 실시간으로 업데이트된다
- [ ] 이름과 비밀번호 입력 후 본인 신청 취소가 완료된다
- [ ] 중복 신청 시 에러 메시지가 표시된다
- [ ] 잘못된 비밀번호로 취소 시도 시 에러 메시지가 표시된다
- [ ] 신청/취소 내역이 노션 데이터베이스에 자동으로 기록된다
- [ ] Vercel 배포 후 정상 동작한다
